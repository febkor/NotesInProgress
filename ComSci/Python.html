Python

Data Model

Everything (int, float, strings, classes, functions …) is an object. No primitive types. All Python objects have:

  • identity: id() corresponds to the memory address of the object
  • type: type()
  • value
    • mutable
      • list
      • set
      • dict
    • immutable
      • int
      • float
      • bool
      • str
      • tuple
      • unicode

The type and identity cannot change during the lifetime of an object.

Python variables are labels/names attached to objects, and not “boxes” of a value. Thus, you can have multiple variables refer to the same object.

Python objects are garbage collected using a reference-counting mechanism.

Reference: https://inventwithpython.com/blog/2018/02/02/the-python-data-model-explained/

Sets

A set is an unordered “bag” of unique values. A single set can contain values of any immutable datatype. Once you have two sets, you can do standard set operations like union, intersection, and set difference. Implemented as a class. 

a_set = {1, 2} 
a_list = ['a', 'hi', True, 42]
a_set = set(a_list) # constructor
# a_set will be unordered

empty = set()

a_set.add(3)
a_set.update({3,4,5}) # adds each element to set
a_set.update([5,6,7]) # same

a_set.discard(6)
a_set.remove(5) # raises `KeyError` exception if the value doesn't exist

a_set.pop() # removes and returns one value
a_set.clear() # empties the set

a_set.union(b_set)  
a_set.intersection(b_set) 
a_set.difference(b_set) # a minus whatever's in b
a_set.symmetric_difference(b_set) # returns elements in exactly one of the sets

a_set.issubset(b_set) 
b_set.issuperset(a_set)

Collections module

The collections module provides specialized data structures.

  • collections.OrderedDict
    • remembers the insertion order of keys
  • collections.defaultdict
    • return default values for missing keys
    • accessing a missing key creates and initializes it
    • dd = collections.defaultdict(list)
    • dd["planets"].append("Mars")
  • collections.ChainMap
    • search multiple dictionaries as a single mapping
    • groups multiple dictionaries into a single mapping
    • lookups search the underlying mappings one by one until a key is found
    • insertions, updates, and deletions only affect the first mapping added to the chain
    • chain = collections.ChainMap(dict1, dict2)

Note that defaultdict is all lower case, and not camel case, for historical reasons.
